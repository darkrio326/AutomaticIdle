# Automatic Idle（暂定名）

一个以生产流程编排与收益优化为核心的放置类游戏原型。玩家通过配置一条循环执行的生产流程（采集 -> 加工 -> 制造 -> 出售/升级/订单），在金币（主动成长）与经验（被动成长）两条成长线之间做权衡，持续提升系统效率。

## 项目目标

- 验证“玩家设计自动化流水线”这一核心玩法是否成立。
- 在最小范围内形成可玩、可优化、可持续的放置体验。
- 构建可扩展的数值/模拟核心（core），为后续版本（桌面端、更多系统）打基础。

## 核心玩法

设计并优化一条自动赚钱的生产流水线。

示例流程：

```text
采矿 x10 -> 熔炼 x5 -> 锻造 x1 -> 出售 x1 -> 循环
```

系统自动计算：

- 单轮耗时 / 收益
- 每秒金币（GPS）
- 资源净变化
- 瓶颈步骤

玩家围绕以下决策进行优化：

- 卖材料 vs 深加工
- 卖出 vs 留作升级/订单
- 刷钱 vs 刷经验
- 优先升级哪个环节

## 主要系统（MVP）

### 1. 流程系统（Flow）

- 列表式步骤链（无分支）
- 顺序执行、按次数循环

### 2. 配方系统（Recipe）

- 采矿 -> 铁矿
- 熔炼 -> 铁锭
- 锻造 -> 铁剑
- 出售 -> 金币

### 3. 资源系统（Resource）

- 金币（货币）
- 铁矿 / 铁锭 / 铁剑（原料 -> 中间品 -> 成品）

### 4. 成长系统（双轨制）

- Gold（主动）：升级设施、解锁内容
- EXP（被动）：动作产经验、提升技能等级、降低耗时

### 5. 升级系统（Upgrade）

- 采矿/熔炼/锻造效率提升（金币 + 材料）

### 6. 订单系统（Order）

- 提交指定物品换取金币/解锁
- 打破单一最优流程

### 7. 模拟器（Simulator）

- 输入流程与配置 -> 输出时间/收益/瓶颈
- 核心逻辑独立于 UI

### 8. 本地存档（Storage）

- localStorage/IndexedDB
- 自动保存与恢复

## 数值与公式（核心约定）

- EXP 获取：采矿 +1，熔炼 +2，锻造 +5
- 升级经验：EXP = 10 x level^1.5
- 技能加成：time *= (1 - level x 2%)
- 总时间：

```text
finalTime = baseTime x (1 - skillBonus) x (1 - upgradeBonus)
```

设计原则：EXP 提供基础效率，Gold 提供爆发优化（乘法叠加）。

## 技术路线

前端：

- Vue 3 + TypeScript + Vite
- 状态管理：Pinia

架构分层：

```text
/src
  /core        # 纯逻辑（无 Vue 依赖）
  /stores      # Pinia 状态
  /components  # UI 组件
  /services    # 存档/工具
  /config      # JSON 配置
```

关键约束：

- 模拟器与 UI 完全解耦
- 所有规则在 core/ 实现
- 配置优先（JSON 驱动）

未来扩展：

- Tauri 2（桌面版）
- 云存档 / 轻后端（可选）

## MVP 成功标准

- 玩家能在 3 分钟内理解玩法
- 会主动调整流程（>=1 次）
- 能感知“卖出 vs 升级”的权衡
- 能持续游玩 15~30 分钟

## 范围控制（当前不做）

- 分支逻辑（if/else）
- 拖拽流程图
- 并行工人
- 战斗/装备/掉落
- 市场波动
- 社交/排行榜

## 开发指南（给自己/AI）

- 先写 core/simulator，再做 UI
- 任何规则改动优先修改 core
- 避免把业务逻辑写进组件
- 每个功能以 IDEA 为单位推进（0.5~2 天）

## 一句话总结

这不是一个“等收益”的放置游戏，而是一个“设计自动化系统”的游戏。

核心在于：让玩家通过优化流程，获得比单纯挂机更强的收益与成长感。
