# 放置生产链游戏设计草案

## 一、项目定义

### 1.1 项目定位

本项目是一款以**生产流程编排与收益优化**为核心玩法的放置类游戏。

玩家通过配置一条循环执行的生产流程，让系统自动完成：

- 资源采集
- 材料加工
- 成品制造
- 出售 / 升级 / 提交订单

并通过持续优化流程结构与资源使用方式，提升整体生产效率与收益。

---

### 1.2 核心体验

与传统 idle 游戏不同，本游戏的核心不在“等待”，而在：

> 设计并优化一条自动赚钱的生产流水线

玩家的乐趣来源于：

- 比较不同流程收益
- 发现瓶颈
- 调整步骤结构
- 权衡资源用途
- 提升系统整体效率

---

### 1.3 核心一句话

> 玩家通过编排自动循环的生产流程，在资源出售、升级与订单之间做出选择，从而逐步构建高效的自动化生产体系。

---

## 二、核心系统结构

### 2.1 流程系统（核心）

例如玩家配置一条循环执行的步骤链：

```
采矿 ×10 → 熔炼 ×5 → 锻造 ×1 → 出售 ×1 → 循环
```

系统自动计算：

- 总耗时
- 总产出
- 每秒收益
- 资源变化
- 瓶颈步骤

---

### 2.2 资源系统

| 类型  | 示例 | 作用    |
| --- | -- | ----- |
| 货币  | 金币 | 通用升级  |
| 原材料 | 铁矿 | 基础生产  |
| 中间品 | 铁锭 | 加工链路  |
| 成品  | 铁剑 | 高价值产出 |

---

### 2.3 成长系统（双轨制）

#### 主动成长（金币）

- 升级设施
- 提高效率
- 解锁内容

#### 被动成长（EXP）

- 执行动作获得经验
- 提升技能等级
- 提供基础效率加成

---

## 三、核心玩法循环

```
流程编排 → 自动生产 → 资源产出 → 卖出/消耗 → 系统提升 → 再优化流程
```

---

## 五、配方系统

### 初始配方

**采矿**

- 输出：铁矿 ×1
- 时间：1秒

**熔炼**

- 输入：铁矿 ×2
- 输出：铁锭 ×1
- 时间：2秒

**锻造**

- 输入：铁锭 ×5
- 输出：铁剑 ×1
- 时间：4秒

**出售铁锭**

- 输出：金币 ×8

**出售铁剑**

- 输出：金币 ×100

---

## 六、经验系统

### 技能

- 采矿
- 熔炼
- 锻造

### EXP获取

- 采矿 +1
- 熔炼 +2
- 锻造 +5

### 升级公式

```
EXP需求 = 10 × 等级^1.5
```

### 效果

```
效率提升 = 等级 × 2%
```

---

## 七、升级系统

### 示例

**采矿升级**

- 成本：金币
- 效果：采矿更快

**熔炉升级**

- 成本：金币 + 铁锭
- 效果：熔炼更快

**锻造升级**

- 成本：金币 + 铁剑
- 效果：锻造更快

---

## 八、订单系统

**订单1**

- 铁锭 ×10
- 奖励：金币 + 解锁锻造

**订单2**

- 铁剑 ×2
- 奖励：金币 + 升级上限

---

## 九、玩家核心决策

- 卖材料还是深加工
- 升级还是卖钱
- 刷钱还是刷经验
- 优先升级哪个环节

## 十二、总结

> 这是一个让玩家通过“设计自动化流程”来获得成长与收益的放置类游戏。

核心不是挂机，而是优化。
玩家的乐趣在于发现更高效的生产链，而不是单纯等待资源积累。
