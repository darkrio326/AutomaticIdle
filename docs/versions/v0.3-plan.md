---

## 1. 版本定位（候选）

- 版本号：v0.3
- 承接版本：v0.2（建筑 / 工具 / 动态订单 / 流程模板已交付）
- 版本主题：稳定性工具 + 系统联动深化
- 目标方向：
  - 先补齐可观测性（Debug 能力），降低后续改动风险；
  - 再增强中长期决策（工具长期投资）；
  - 最后处理系统耦合（订单与主流程联动），让策略选择更有压力。

## 5. In Scope / Out of Scope（候选口径）

### In Scope（优先）

- Debug 面板 + Runtime 测试（稳定性基线）
- 调优脚本（开发效率基线）
- 工具长期投资机制（轻量版）
- 订单与主流程的弱耦合联动

### Out of Scope（本版默认不做）

- 全功能图形化流程编辑器（节点拖拽 + 连线编辑 + 完整回写）
- 大规模新资源链扩展（铜/钢之外再开新资源族）
- 账号云存档与联机系统

---

## 6. 关键风险与控制

| 风险 | 触发点 | 控制策略 |
|---|---|---|
| Debug 工具侵入主流程 | 调试逻辑混入 runtime 主路径 | debug 分支隔离，默认关闭，只在 query 开启 |
| 测试补齐成本失控 | 缺少稳定可测边界 | 先锁定 4 类核心行为，再逐步扩展 |
| 工具机制过重影响易玩性 | 耐久/维护链条过复杂 | 仅做“轻量维护成本”版本，不做多层维修树 |
| 订单耦合导致体验割裂 | 强制打断玩家当前链路 | 采用引导型联动，不强行中断运行中步骤 |

---

## 7. 候选版门禁（建议）

1. `npm run build` 通过。
2. `npx vue-tsc --noEmit` 通过。
3. Runtime 回归用例通过（至少覆盖 4 类核心行为）。
4. Debug 面板默认关闭，不影响普通玩家路径。
5. 订单/工具新增规则不会让主循环出现“运行中但不产出”退化。
