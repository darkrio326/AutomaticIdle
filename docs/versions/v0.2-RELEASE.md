## 4. 本版交付摘要

- 建筑系统：新增建筑购买、解锁资源/配方链路与状态持久化，形成明确的进度门槛
- 工具系统：新增工具购买、配方耗时加速、实时引擎接入与持久化恢复
- 动态订单：新增 3 槽位随机订单、时限/冷却/删除/奖励链路与新 UI 展示
- 流程模板：支持最多 5 套模板的保存、加载、删除与 localStorage 持久化
- 欢迎页与路由：新增欢迎页、`vue-router` 路由切换、游戏页拆分
- 发布质量修复：修复 Set 序列化导致引擎假运行、getter 调用错误、工具效果未同步、快速配方进度条异常、暗色主题亮色降级变量错用、购买提示位置错误等问题

## 5. 发布结论

- 结论：PASS
- 阻断原因：无
- 条件发布说明：`Runtime 调试面板` 未纳入本次发布，顺延至 v0.3 处理

## 6. 回滚方案

- 触发条件：主循环无法运行、购买状态无法持久化、订单系统无法刷新、模板系统破坏现有流程编辑
- 回滚步骤：按功能提交粒度执行 `git revert`；优先回退最近的 UI/交互修复提交，再视情况回退欢迎页/路由或订单系统提交
- 影响评估：回滚 UI 修复影响较小；回滚订单/工具/建筑相关提交将直接影响 v0.2 主功能范围

## 7. 发布后观察

- 观察窗口：发布后首轮 1 ~ 2 天本地试玩/验证
- 关键指标：主循环持续运行稳定性、订单刷新是否正常、建筑/工具状态持久化、快速配方进度条表现、欢迎页到游戏页路由切换是否稳定
- 异常登记与处理：若出现“运行中但不产出”、购买后状态丢失、订单卡死或 UI 配色回退，优先登记为 P0/P1 并以热修复处理
