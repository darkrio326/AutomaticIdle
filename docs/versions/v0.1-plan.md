# v0.1 版本计划

更新日期：2026-03-30

---

## 1. 版本定位

- **版本号**：v0.1
- **承接版本**：— （v0.1 为首个正式版本，基于 MVP 原型迭代 001～013 完成后启动）
- **版本主题**：从"流程收益计算器"升级为"真正有运行感的放置游戏"
- **核心目标**：  
  实现 runtime 实时引擎，让流程在界面上持续循环运转；玩家能看到当前正在执行哪个步骤、步骤进度如何、资源随时间持续增长。同步完成工程 UI 向 prototype 原型的对齐收敛，确保达到可发布的体验基线。

## 7. 验收与回滚

### 验收标准

1. **运行感**：打开页面后流程自动开始循环，当前步骤有高亮和进度条；不需要手动点击"运行"配置即可看到状态持续变化。
2. **资源反馈**：金币、矿石、铁锭、铁剑等资源数值在运行中可见持续变化，不是静态数字。
3. **修改生效**：玩家修改流程步骤后，当前动作完成后自动切入新流程，无跳步、无状态丢失。
4. **存档连续**：存档/恢复逻辑在 runtime 引入后仍正常工作；离线收益不因 runtime 引入而失效。
5. **UI 可读**：页面信息层次清晰，关键指标（当前步骤/资源/GPS/技能/升级/订单）布局合理，参考 prototype 视觉基线。
6. **类型检查通过**：`vue-tsc --noEmit` 无报错。
7. **构建通过**：`npm run build` 成功，dist/ 可访问。

### 回归范围

| 模块 | 验证点 |
|------|--------|
| 流程编辑 | 步骤增删改后下一轮正确生效 |
| 资源结算 | 每次动作完成后资源变化正确 |
| 技能升级 | EXP 积累到阈值后升级生效 |
| 升级购买 | 金币足够时购买升级成功；流程效率即时更新 |
| 订单提交 | 库存满足条件后提交成功；奖励正确发放 |
| 存档恢复 | 刷新后流程、库存、技能、升级、订单状态完整恢复 |
| 离线收益 | 关闭一段时间后重开有正确补发；不重复领取 |

### 回滚策略

- 每个 ITER 单独提交，可按 commit hash 逐步回退
- Phase 1（引擎层）改动不直接影响当前 UI 展示行为，若 Phase 1 引入问题，Phase 3 代码未合并前 UI 仍保持静态模拟模式
- `runtimeStore.ts` 新建文件，不修改 `flowStore.ts` 核心结算入口，降低回滚时的影响范围；ITER-018 起与 flowStore 清晰划定边界
